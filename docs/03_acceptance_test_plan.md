# 受け入れテスト計画

## 目的

このテストは、アプリが「動くデモ」ではなく、Suno音源を安全に整えてWAVマスターを書き出せる作業台になっているかを判断するためのものです。

合格条件は、音が良くなると断言することではありません。少なくとも、壊さない、古い結果を出さない、出力値を確認できる、WAVを主成果物として保存できる、という実用上の土台を満たすことです。

## 必須テスト音源

最低限、次の種類を確認します。

- Sunoの普通のポップス
- 低域が強い曲
- 高域が刺さる曲
- すでに音圧が高い曲
- 静かなイントロを含む曲
- 10分前後の長尺音源
- WAV入力
- MP3入力

今回の実音源検査では、ユーザー提供の5曲を使用しました。

- `Tail Light.wav`
- `シャッター.wav`
- `線香花火のプロミス.wav`
- `恋色万華鏡.wav`
- `残光に踊る.wav`

検査結果は [05_real_audio_test_report.md](05_real_audio_test_report.md) に記録します。

## 読み込み

### A-01 WAV読み込み

合格条件:

- ファイルを選ぶと波形が表示される
- Beforeを再生できる
- Afterが自動生成される
- After生成が終わるまで書き出しできない
- 読み込み失敗時にアプリが固まらない

### A-02 MP3/FLAC読み込み

合格条件:

- ブラウザがデコードできる形式は読み込める
- 最終出力はWAV推奨のまま表示される
- 読み込めない場合は理由が表示される

## 比較再生

### B-01 Before/After比較

合格条件:

- Before/Afterを切り替えて同じ位置から聴ける
- 音量マッチをオン/オフできる
- Afterが単に大きいだけで良く聴こえる状態を避けられる

### B-02 シーク

合格条件:

- 波形クリックで再生位置を移動できる
- 再生中のシークで破綻しない

## 状態管理

### C-01 連続操作

合格条件:

- スライダーを素早く動かしても、最後の設定でAfterが生成される
- 古い処理結果が後から上書きされない
- 再生成待ちの間は書き出しできない

### C-02 読み込み中の保護

合格条件:

- 新しいファイル読み込み中に古いAfterを書き出せない
- 古い精密測定値が残り続けない

## 音声処理

### D-01 基本補正

合格条件:

- 極端な低域の膨らみが軽くなる
- 音が明らかに薄くなりすぎない
- 基本補正オン/オフの差が聴き取れる

### D-02 直感スライダー

合格条件:

- 明るさ、太さ、迫力、広がり、なめらかさの変化が直感と大きくズレない
- 上げ下げしても即座に破綻しない
- やりすぎた場合に耳で気づける範囲に収まる

### D-03 リミッター

合格条件:

- 書き出し音源でクリップしない
- 迫力を上げても極端な歪みを出しにくい
- ffmpeg精密仕上げ後のTrue Peakが目標以下になる

## 書き出し

### E-01 Master WAV

条件:

- ffmpeg接続済みの場合は `/api/finalize` を使う
- ffmpeg未接続の場合はブラウザ内WAV書き出しにフォールバックする

合格条件:

- 24bit WAVを書き出せる
- 16bit WAVを書き出せる
- Integrated LUFSが目標からおおむね±0.5 LU以内
- True Peakが-1.0 dBTP以下
- UIに書き出し後の精密値が表示される
- ファイル名に `master` が入るのはWAVだけ

### E-02 MP3コピー

条件:

- ffmpeg接続済みの場合だけ有効
- `/api/finalize` のMP3モードを使う

合格条件:

- 320kbps MP3を書き出せる
- ファイル名は `copy-320k.mp3`
- UI上で「マスター」ではなく「コピー」として扱われる
- 書き出し後のLUFS/True Peakが再測定される

## 性能

### F-01 一般的な3から5分程度のWAV

目安:

- After生成: 10秒以内を目標
- ffmpeg精密仕上げ: 60秒以内を目標

### F-02 長尺・大容量

合格条件:

- 10分超、大容量、高サンプルレートでは警告が表示される
- 処理中であることが分かる
- 失敗しても復帰できる

## 回帰テスト

修正ごとに最低限確認します。

- `node --check app.js`
- `python -m py_compile backend/server.py`
- `/api/health`
- `/api/analyze`
- `/api/finalize` WAV
- `/api/finalize` MP3
- ブラウザ初期表示
- ffmpeg接続済み表示

## リリース判定

実用ベータとして出せる条件:

- P0の未達がない
- 実音源でWAV/MP3の精密仕上げが通る
- 古いAfterを書き出せない
- WAVが主成果物、MP3がコピーとして明確に分かれている
- 残る未達がユーザーに説明済みの制限である
