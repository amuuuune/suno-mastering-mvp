# 要件定義

## 1. コンセプト

SunoなどのAI音楽生成サービスで作った完成済みステレオ音源を、音楽知識が少ないユーザーでも、耳で比較しながら安全に仕上げられるマスタリング作業台にする。

このアプリは「プロのマスタリングエンジニアの完全代替」ではない。目的は、ユーザーが意図を選び、アプリが測定、比較、基本補正、書き出しの安全性を支えること。

## 2. 対象ユーザー

- Suno、Udioなどで曲を作っている個人ユーザー。
- DAWやプラグインの専門知識は薄い。
- EQ、コンプ、リミッターの数値より、「明るさ」「太さ」「迫力」「広がり」「なめらかさ」で触りたい。
- 最終的にSNS、動画、配信、保管用に音源を整えたい。
- 自分の耳でBefore/Afterを選びたい。

## 3. 対象音源

- 完成済みの1ファイル音源。
- 主な長さは2分から8分程度。
- 入力はWAV、MP3、FLACなど、ブラウザまたはffmpegが読める形式。
- モノラルまたはステレオ。
- マルチトラック、ステム、ボーカル分離、ミックス修正は対象外。

## 4. 成功条件

実用ベータとして成功とみなす条件は次の通り。

- 音源を読み込める。
- Before/Afterを音量マッチして比較できる。
- 基本補正を安全にオン/オフできる。
- 直感スライダーで音の方向性が分かる程度に変わる。
- 書き出し前にAfterが最新設定と一致している。
- WAVを主出力として保存できる。
- ffmpeg接続時、WAV書き出しは2パスloudnormで目標LUFSへ寄せる。
- True Peakが目標上限を超える場合は、書き出し側で抑えるか、失敗としてユーザーに分かる形で止める。
- MP3はマスターではなく任意の圧縮コピーとして扱う。
- 処理中、再生成待ち、書き出し中に誤操作で古い音源を書き出せない。

## 5. 非目標

v1実用ベータでやらないこと。

- プロエンジニアの判断の完全自動化。
- VST/AU/AAXプラグイン化。
- DAW内リアルタイムプラグイン動作。
- マルチトラックミックス。
- ピッチ補正、ボーカル抽出、ノイズ除去特化。
- 楽曲生成。
- 市販マスタリングスイートと同等の全機能。

## 6. 品質レベルの定義

### MVP

音が読み込め、触れ、比較でき、WAVを書き出せる。測定やDSPは簡易でもよいが、ユーザーにその範囲を明示する。

### 実用ベータ

Suno音源を複数曲処理しても破綻しにくい。WAVマスター出力はffmpeg精密仕上げを通す。状態管理と誤書き出し防止が入っている。

### プロ品質

市販プラグイン相当のリミッター、マルチバンド処理、ダイナミックEQ、リアルタイムメーター、厳密な検品、長時間安定動作を満たす。現時点の約束範囲ではない。

## 7. 最優先の原則

- WAVが主出力。
- MP3は任意コピー。
- 数値より耳が優先。ただし書き出し安全性は数値で守る。
- ブラウザ内メーターは作業目安。
- ffmpegメーターは書き出し検品用。
- ユーザーの判断を奪わず、失敗しやすい部分をアプリが防ぐ。

## 8. 「使えない」と判断する条件

以下のどれかに該当する場合、実用ベータとして不合格。

- 設定変更後に古いAfterを書き出せる。
- 処理中に書き出しできる。
- WAVが主出力だと分からない。
- MP3がマスター扱いに見える。
- Before/After比較が音量差だけで良く感じる状態になっている。
- ffmpeg接続済みなのに精密仕上げを通さずWAVを書き出す。
- 書き出し後のLUFS/True Peakが確認できない。
- 失敗時にユーザーが原因を推測できない。
- 長尺または重い音源でフリーズしたように見え、状態表示がない。
