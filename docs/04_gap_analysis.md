# 現状差分と未達リスト

更新日: 2026-05-05

## 判定

現時点で「これがないと危なくて使えない」レベルのP0未達はありません。

今回の修正と検査で、次の実用上の穴を閉じました。

- WAVを主成果物、MP3を任意の圧縮コピーとしてUIとファイル名で分離
- ffmpegをプロジェクト内 `tools/ffmpeg/ffmpeg.exe` から検出
- WAV/MP3の両方を `/api/finalize` で2パス `loudnorm` 仕上げ
- 書き出し後のLUFS/True Peak/LRAをレスポンスヘッダーで返し、UIへ表示
- 古いAfter、読み込み中、処理中、解析中、書き出し中の誤書き出しを防止
- 10分超、大容量、高サンプルレート音源への警告
- スペクトラムを中央1点ではなく全体平均ベースへ改善
- ルックアヘッドリミッターと動的なめらかさ処理を追加
- 実音源5曲でWAV/MP3の精密仕上げを検査
- 古い非精密MP3 API `/api/export-mp3` を削除

## 解消済みP0

### 古いAfterを書き出せる可能性

状態: 解消済み。

対応:

- `processVersion` で古い非同期処理結果を破棄
- `needsProcess` で再生成待ちを明示
- `isProcessing` / `isAnalyzing` / `isExporting` で操作を制限
- 読み込み中と処理中はWAV/MP3書き出しボタンを無効化

### MP3がマスターに見える問題

状態: 解消済み。

対応:

- UIの主ボタンを `Master WAV` に固定
- MP3は `MP3コピー` として精密エンジン側へ移動
- MP3ファイル名を `copy-320k.mp3` に変更
- 古い `/api/export-mp3` を削除し、MP3も `/api/finalize` 経由に統一

### 精密LUFS/True Peakがない問題

状態: 実用ベータとして解消済み。

対応:

- ffmpeg `loudnorm` でIntegrated LUFS、True Peak、LRAを取得
- 書き出し時に2パス `loudnorm` で目標LUFS/True Peakへ寄せる
- WAV/MP3書き出し後の再測定値をUIに表示

注意:

外部納品前の最終検品まで完全に保証するものではありません。厳密な納品検品では、別メーターや別環境でのクロスチェックが必要です。

## 解消済みP1

### 長尺・大容量音源で何が起きているか分からない

状態: 解消済み。

対応:

- ファイル長、サイズ、サンプルレートを表示
- 10分超、大容量、高サンプルレートへの警告
- 処理中、解析中、書き出し中の状態表示

### 実音源での受け入れ検査がない

状態: 機械検査は解消済み。

対応:

- ユーザー提供の5曲で `/api/analyze` と `/api/finalize` を実行
- WAV/MP3とも目標 -14 LUFS付近、True Peak -1 dBTP以下を確認
- 結果を [05_real_audio_test_report.md](05_real_audio_test_report.md) に記録

残る確認:

耳で聴いたときの好み、プリセットの効き方、曲ごとの「狙いに合うか」はユーザー確認が必要です。

## 残る未達

### P1: 主観リスニング調整

内容:

実音源5曲で機械検査は通っていますが、Natural/Bright/Warm/Loud/Vocal/Wide/Smoothのプリセット値が本当に気持ちいいかは、耳で聴いて調整する必要があります。

扱い:

これはアプリの安全性ではなく音作りの品質問題です。ユーザーが数パターン聴き比べて「この方向が良い/悪い」を返した段階で詰めます。

### P1: 外部納品レベルの独立検品

内容:

ffmpegの `loudnorm` による値は実用上かなり頼れますが、商用納品で絶対値を保証するには別系統のメーター、DAW、ストリーミング変換後の確認が必要です。

扱い:

現状はSuno音源の個人制作、SNS公開、配信用の実用ベータです。納品検品ツールとは呼びません。

### P2: ブラウザ内処理のメモリ限界

内容:

After生成はブラウザの `AudioBuffer` 上で行うため、長尺・高サンプルレート・巨大ファイルでは重くなる可能性があります。

現状対応:

警告と状態表示は実装済みです。

将来対応:

重い処理をPython/ffmpeg側へ寄せ、ストリーミング処理にする。

### P2: リファレンス曲比較

内容:

「この曲みたいに寄せる」ためのリファレンス比較は未実装です。

扱い:

現行の目的は、まずWAVマスターを安全に作ることです。リファレンス機能は次段階です。

### P3: 便利機能

候補:

- プリセット保存
- 複数案の書き出し
- 処理履歴
- デスクトップアプリ化
- VST/AUプラグイン化
- マルチバンドコンプレッサー
- より本格的なダイナミックEQ
- 変換後MP3/AAC/Opusのラウドネス比較

## 現在の結論

「穴が多すぎて使えない」状態からは抜けました。

ただし、「プロの判断を自動で代替するツール」ではありません。現状の正しい位置づけは、Suno音源を直感的に調整し、WAVマスターとして安全に保存し、ffmpegで精密値を確認できる実用ベータです。
